## 第一步：能连上后端（5分钟）

### 目标：用 JavaScript 调用后端的 `/api/health` 接口

在上面 `<script>` 标签里写：

```javascript
// 发起请求
fetch('http://localhost:8080/api/health')
  .then(response => response.json())  // 把响应变成 JSON
  .then(data => {
    console.log('后端返回:', data);
    document.getElementById('result').innerHTML = 
      '✅ 连接成功！后端状态: ' + data.data.status;
  })
  .catch(error => {
    console.error('❌ 连接失败:', error);
    document.getElementById('result').innerHTML = '❌ 连接失败，检查后端是否启动';
  });
```

**刷新浏览器**，你会看到：
- 页面显示 "✅ 连接成功！后端状态: UP"
- F12 控制台打印 `后端返回: {code: 200, ...}`

**恭喜！你已经"连上"后端了。** `fetch` 就是发 HTTP 请求的方法。

## 第二步：能要到数据（10分钟）

### 目标：获取所有建筑列表（63个）

把刚才的代码改成这样：

```javascript
// 调用"建筑列表"接口
fetch('http://localhost:8080/api/navigation/buildings')
  .then(response => response.json())
  .then(data => {
    console.log('建筑数据:', data);
    
    // 后端返回格式：{code: 200, data: [...]}
    if (data.code === 200) {
      const buildings = data.data;  // 这是真正的数据（数组）
      
      // 在页面上显示前5个建筑
      let html = '<h2>建筑列表（前5个）</h2><ul>';
      buildings.slice(0, 5).forEach(b => {
        html += `<li>${b.name} (ID: ${b.id})</li>`;
      });
      html += '</ul>';
      
      document.getElementById('result').innerHTML = html;
    }
  })
  .catch(error => {
    console.error('请求失败:', error);
  });
```

**刷新浏览器**，你会看到：
```
建筑列表（前5个）
• 校卫生所 (ID: 1001)
• 花房 (ID: 1002)
• 第二食堂 (ID: 1003)
• ...
```

**关键点**：
- 后端**所有接口**都返回 `{code, message, data}` 这个格式
- 真正的数据在 `data.data` 里（第一个 data 是 fetch 解析的，第二个 data 是后端包裹的）
- `code === 200` 表示成功

## 第三步：能登录拿Token（15分钟）

### 为什么要 Token？

后端有些接口（比如"获取我的个人信息"）需要**证明你是谁**，不能随便谁都能调。  
**Token 就是你的"身份证"**，登录后后端给你一个，以后每次请求都带上它。

### 3.1 先调登录接口拿到 Token

```javascript
// 用测试账号登录
fetch('http://localhost:8080/api/auth/login', {
  method: 'POST',  // 登录要用 POST
  headers: {
    'Content-Type': 'application/json'  // 告诉后端：我发的是 JSON
  },
  body: JSON.stringify({
    studentId: 'admin',       // 测试账号
    password: 'admin123'
  })
})
  .then(response => response.json())
  .then(data => {
    console.log('登录返回:', data);
    
    if (data.code === 200) {
      const token = data.data.token;  // 这就是你的"身份证"
      console.log('拿到Token:', token);
      
      // 保存起来（存到 localStorage，浏览器关了也不丢）
      localStorage.setItem('token', token);
      
      document.getElementById('result').innerHTML = 
        '✅ 登录成功！Token已保存<br>Token前20位: ' + token.substring(0, 20) + '...';
    } else {
      alert('登录失败: ' + data.message);
    }
  });
```

**刷新浏览器**，你会看到：
```
✅ 登录成功！Token已保存
Token前20位: eyJhbGciOiJIUzI1NiJ9...
```

**Token 长这样**：`eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIx...`（很长一串）

### 3.2 用 Token 调用"需要登录"的接口

现在你有 Token 了，试试调用"获取我的信息"：

```javascript
// 从 localStorage 取出之前保存的 Token
const token = localStorage.getItem('token');

if (!token) {
  alert('还没登录！先运行上面的登录代码');
} else {
  // 带着 Token 请求
  fetch('http://localhost:8080/api/user/me', {
    headers: {
      'Authorization': 'Bearer ' + token  // 关键！把 Token 放请求头里
    }
  })
    .then(response => response.json())
    .then(data => {
      console.log('我的信息:', data);
      
      if (data.code === 200) {
        const user = data.data;
        document.getElementById('result').innerHTML = 
          `<h2>我的信息</h2>
           <p>学号: ${user.studentId}</p>
           <p>昵称: ${user.username}</p>
           <p>角色: ${user.role}</p>`;
      } else if (data.code === 401) {
        alert('Token过期了，重新登录吧');
      }
    });
}
```

**刷新浏览器**（如果之前登录过，localStorage 里有 Token），你会看到：
```
我的信息
学号: admin
昵称: 系统管理员
角色: ADMIN
```

**如果没 Token 或 Token 错了**，后端返回 `{code: 401}`，表示"你没登录"。

现在你已经会：
1. ✅ **发请求** (`fetch`)
2. ✅ **拿数据** (解析 `{code, data}`)
3. ✅ **登录** (拿 Token)
4. ✅ **带 Token 访问** (放 `Authorization` 请求头)

## 🤔 常见问题

### Q1: 点按钮没反应
**答**: 按 F12 看控制台（Console）有没有报错。常见错误：
- `CORS error` → 后端没启动，或者地址写错了
- `Failed to fetch` → 检查 `http://localhost:8080` 是否能访问

### Q2: 登录返回 `{code: 400, message: "学号或密码错误"}`
**答**: 测试账号可能不存在。联系后端同学确认测试账号，或者先调注册接口创建一个。

### Q3: Token 是什么？为什么要用它？
**答**: 
- 后端是**无状态**的，不记住你。每次请求它都不知道你是谁。
- Token 就像门禁卡，登录时后端发给你，以后每次刷卡（带 Token）它才认识你。
- Token 格式：`Authorization: Bearer <一长串字符>`

### Q4: 所有接口都要 Token 吗？
**答**: 不是。
- **不要 Token**: 健康检查、建筑列表、寻路、搜索（公开信息）
- **要 Token**: 获取我的信息、修改资料、发布失物、管理后台

试一下就知道：不带 Token 调 `/api/user/me`，后端返回 `{code: 401}`。

你已经学会"怎么和后端对话"了。接下来：

1. **用 Vue/React 替代这个 HTML**  
   现在你用的是纯 HTML + `fetch`，实际项目会用框架（Vue 3 / React）。但**原理一模一样**：

   - 也是存 Token、带 Token

2. **把这四步封装成工具函数**  
   每次都写 `fetch(...).then(...).then(...)` 太麻烦，封装成：
   ```javascript
   // 统一的请求函数
   function request(url, options = {}) {
     const token = localStorage.getItem('token');
     if (token) {
       options.headers = options.headers || {};
       options.headers.Authorization = 'Bearer ' + token;
     }
     return fetch(API_BASE + url, options).then(res => res.json());
   }

   // 使用
   request('/api/user/me').then(data => { ... });
   ```

3. **画 Canvas 地图**  
   拿到建筑数据后，用 `<canvas>` 把它画出来：
   ```javascript
   const canvas = document.getElementById('map');
   const ctx = canvas.getContext('2d');
   buildings.forEach(b => {
     ctx.beginPath();
     ctx.arc(b.x, b.y, 5, 0, Math.PI * 2);
     ctx.fill();
   });
   ```

4. **参考"前端开发接入指南.md"**  
   项目根目录有完整的接入文档，里面有 Vue 3 完整代码、路由守卫、状态管理。

# 附录：本地部署后端服务（从零把后端跑起来）

> 如果访问 `http://localhost:8080/api/health` 打不开，说明后端没启动。
> 跟着这个附录，**你自己也能把后端跑起来**，不用等后端同学。

后端是一个 **Java 程序 + MySQL 数据库**。要跑起来需要三步：
```
装环境（JDK + Maven + MySQL）→ 初始化数据库 → 启动后端
```

---

## 第 0 步：装三样东西

后端依赖这三个，**先确认装没装**。打开命令行（Windows 按 Win+R 输入 `cmd`，或用 Git Bash），逐条运行：

```bash
java -version     # 要看到 17.x.x
mvn -version      # 要看到 3.x.x
mysql --version   # 要看到 8.0.x
```

**哪个报错"不是内部命令 / command not found"，就装哪个：**

| 工具 | 作用 | 下载地址 |
|------|------|----------|
| **JDK 17** | 运行 Java 程序 | https://adoptium.net/temurin/releases/?version=17 （选 Windows x64 MSI，安装时勾选 "Add to PATH" 和 "Set JAVA_HOME"） |
| **Maven** | 构建/启动 Java 项目 | https://maven.apache.org/download.cgi （下载 bin.zip，解压后把 `bin` 目录加到系统 PATH） |
| **MySQL 8.0** | 数据库 | https://dev.mysql.com/downloads/installer/ （选大的离线安装包，Server only 即可） |

> ⚠️ 装完**关掉命令行重开**，再运行上面三条命令确认。看到版本号才算装好。

---

## 第 1 步：初始化数据库（只需做一次）

数据库刚装好是空的，要把表和数据灌进去。

### 1.1 确认 MySQL 服务在运行

Windows 按 `Win+R` → 输入 `services.msc` → 找到 `MySQL80` → 状态应该是"正在运行"。
没运行就右键"启动"。

### 1.2 执行两个 SQL 脚本

在项目根目录（`campus-backend` 的上一层）打开命令行，运行：

```bash
# 第 1 个脚本：建库、建表、建账号、初始数据
mysql -u root -p --default-character-set=utf8mb4 < sql/init.sql

# 第 2 个脚本：导入地图数据（183个节点 + 235条边）
mysql -u root -p --default-character-set=utf8mb4 campus < sql/map_data.sql
```

- 每条命令回车后会让你**输入 MySQL 的 root 密码**（装 MySQL 时你自己设的）
- `--default-character-set=utf8mb4` 不能省，否则中文会乱码报错
- **顺序不能反**：先 `init.sql` 再 `map_data.sql`

### 1.3 验证数据进去了

```bash
mysql -u campus_user -pcampus123 campus -e "SELECT COUNT(*) FROM map_node;"
```

看到 `183` 就对了（`campus_user` / `campus123` 是脚本自动创建的应用账号）。

## 🆘 部署常见问题

### Q: `mvn spring-boot:run` 报 "Failed to obtain JDBC Connection"
**原因**: 后端连不上数据库。检查：
1. MySQL 服务是否在运行（`services.msc` 看 `MySQL80`）
2. 有没有执行第 1 步的 SQL 脚本（数据库/账号是否存在）
3. 运行验证命令 `mysql -u campus_user -pcampus123 campus -e "SELECT 1;"` 能不能连上

### Q: SQL 脚本报 "Data too long" 或中文乱码
**原因**: 没加 `--default-character-set=utf8mb4`。重新执行，别漏这个参数。

### Q: `mvn` 或 `java` 提示"不是内部命令"
**原因**: 环境变量没配好。重装时勾选"Add to PATH"，或手动把安装目录的 `bin` 加到系统 PATH，然后**重开命令行**。

### Q: 端口 8080 被占用
**现象**: 启动报 "Port 8080 was already in use"
**解决**: 可能后端已经在跑了（直接访问 health 试试）。或者找到占用进程杀掉：
```bash
netstat -ano | findstr :8080     # 看占用端口的进程 PID
taskkill /F /PID <上面查到的PID>   # 杀掉它
```

### Q: 我不想每次都开命令行跑 mvn，有别的办法吗？
**答**: 有。先打包成一个 jar，以后双击/一条命令就能跑：
```bash
cd campus-backend
mvn clean package -DskipTests          # 打包（生成 target/campus-system-0.0.1-SNAPSHOT.jar）
java -jar target/campus-system-0.0.1-SNAPSHOT.jar   # 启动
```
