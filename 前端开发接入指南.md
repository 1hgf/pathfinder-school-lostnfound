## 🔑 认证流程实现

### 1. Axios 拦截器配置

```javascript
// src/utils/request.js
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'
import router from '@/router'

const service = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 10000
})

// 请求拦截器：自动带 Token
service.interceptors.request.use(
  config => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器：统一错误处理
service.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      ElMessage.error(res.message || '请求失败')
      // 401 跳转登录页
      if (res.code === 401) {
        useUserStore().logout()
        router.push('/login')
      }
      return Promise.reject(new Error(res.message))
    }
    return res.data  // 直接返回 data 字段
  },
  error => {
    ElMessage.error(error.message || '网络错误')
    return Promise.reject(error)
  }
)

export default service
```

### 2. 用户状态管理（Pinia）

```javascript
// src/stores/user.js
import { defineStore } from 'pinia'
import { ref } from 'vue'
import request from '@/utils/request'

export const useUserStore = defineStore('user', () => {
  const token = ref(localStorage.getItem('token') || '')
  const userInfo = ref(null)

  // 登录
  async function login(studentId, password) {
    const data = await request.post('/api/auth/login', { studentId, password })
    token.value = data.token
    userInfo.value = {
      userId: data.userId,
      studentId: data.studentId,
      username: data.username,
      role: data.role
    }
    localStorage.setItem('token', data.token)
  }

  // 登出
  function logout() {
    token.value = ''
    userInfo.value = null
    localStorage.removeItem('token')
  }

  // 获取用户信息
  async function getUserInfo() {
    const data = await request.get('/api/user/me')
    userInfo.value = data
  }

  return { token, userInfo, login, logout, getUserInfo }
})
```

### 3. 路由守卫

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import { useUserStore } from '@/stores/user'

const routes = [
  { path: '/login', component: () => import('@/views/Login.vue') },
  { path: '/register', component: () => import('@/views/Register.vue') },
  {
    path: '/',
    component: () => import('@/views/Layout.vue'),
    meta: { requiresAuth: true },
    children: [
      { path: '', redirect: '/map' },
      { path: 'map', component: () => import('@/views/Map.vue') },
      { path: 'lostfound', component: () => import('@/views/LostFound.vue') },
      { path: 'profile', component: () => import('@/views/Profile.vue') }
    ]
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// 路由守卫
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  if (to.meta.requiresAuth && !userStore.token) {
    next('/login')
  } else {
    next()
  }
})

export default router
```

## 📡 完整 API 文档

### 认证接口（无需 Token）

| 接口 | 方法 | 说明 | 请求参数 | 响应示例 |
|------|------|------|----------|----------|
| `/api/auth/register` | POST | 注册 | `{studentId, username, password, email?, phone?}` | `{"code":200, "message":"注册成功"}` |
| `/api/auth/login` | POST | 登录 | `{studentId, password}` | `{"code":200, "data":{"token":"...", "userId":1, "role":"USER"}}` |

**错误码**:
- `400`: 参数校验失败、业务错误（如学号已存在、密码错误）
- `401`: 未登录或 Token 过期
- `403`: 无权限
- `500`: 服务器内部错误

---

### 用户接口（需带 Token）

| 接口 | 方法 | 说明 | 请求头 | 响应示例 |
|------|------|------|--------|----------|
| `/api/user/me` | GET | 获取个人信息 | `Authorization: Bearer <token>` | `{"code":200, "data":{"userId":1, "studentId":"2024001", "username":"张三", "phone":"****1234"}}` |
| `/api/user/profile` | PUT | 修改资料 | 同上 | `{"code":200}` |
| `/api/user/password` | PUT | 修改密码 | 同上，Body: `{oldPassword, newPassword}` | `{"code":200}` |

---

### 导航接口（公开）

| 接口 | 方法 | 说明 | 请求参数 | 响应示例 |
|------|------|------|----------|----------|
| `/api/navigation/buildings` | GET | 所有建筑 | 无 | `{"code":200, "data":[{id, name, x, y, type}]}` |
| `/api/navigation/search` | GET | 搜索节点 | `?keyword=图书` | 同上 |
| `/api/navigation/route` | POST | 路径规划 | `{startNodeId, endNodeId}` | `{"code":200, "data":{"path":[...], "totalDistance":582.91, "nodeCount":8}}` |
| `/api/navigation/nearest` | GET | 坐标吸附 | `?x=500&y=300` | `{"code":200, "data":{id, name, x, y}}` |

**注意**:
- `route` 返回的 `path` 是节点 ID 数组，前端需根据 ID 查询各节点坐标后绘制
- `totalDistance` 单位是米

## 🐛 常见问题

### Q1: 跨域问题
**现象**: 浏览器报 CORS 错误  
**原因**: 后端已配置 CORS 允许所有来源，如果仍报错，检查：
- 前端是否用 `http://localhost:5173`（Vite 默认端口）
- 后端是否正常启动在 `http://localhost:8080`

### Q2: Token 失效
**现象**: 突然所有请求返回 401  
**原因**: JWT Token 有效期 24 小时，过期后需重新登录  
**解决**: 响应拦截器捕获 401 后自动跳转登录页（见上文代码）

### Q3: 路径绘制不连续
**现象**: Canvas 上路径断开或错位  
**原因**: `path` 数组中的节点 ID 需要查询对应坐标，可能某些节点未加载  
**解决**: 
```javascript
// 确保所有节点都在 nodesMap 中
const pathNodes = pathIds.map(id => nodesMap.get(id)).filter(Boolean);
```

### Q4: 中文乱码
**现象**: 某些 API 返回的中文显示乱码  
**原因**: 这是后端开发环境 (Windows bash) curl 测试时的显示问题，实际数据库和响应编码正确  
**验证**: 浏览器 Network 面板查看 Response，应该显示正常中文
