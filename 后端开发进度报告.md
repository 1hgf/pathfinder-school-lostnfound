**🐛 关键 Bug 修复**:

**问题**: HTML 版本的 Dijkstra 实现使用"距离的平方"作为边权：
```javascript
// 错误实现
navDist2 = dx*dx + dy*dy  // 没有开根号
```

**后果**: 因为 `a² + b² ≠ (a+b)²`，平方距离不满足可加性，导致算法计算的"最短路径"并非真实最短距离。

**修正**: Java 版本使用真实欧氏距离：
```java
// 正确实现
private double calculateDistance(MapNode a, MapNode b) {
    double dx = a.getX() - b.getX();
    double dy = a.getY() - b.getY();
    return Math.sqrt(dx * dx + dy * dy);  // 真实距离
}
```

**注意**: 坐标吸附（`findNearestNode`）仍可用平方距离，因为单次比较不涉及累加。

**🚴 骑行模式设计决策**:

**数据事实**:
- 所有 63 个建筑节点**仅通过 PATH 边连接**，不在任何 ROAD 边中
- 如果骑行模式简单排除 PATH 边，所有建筑会变成孤岛，无法寻路

**决策**: 骑行模式暂时**搁置开发**，待前端接入后根据实际需求优化。可能的方案：
1. 骑行模式使用全图，但对 PATH 边加权重惩罚（×1.5）
2. 骑行只在路口间寻路，建筑"最后一公里"提示步行
